## 2. 整体机制架构（T2 Mechanism Architecture）

### 2.1 底层：资产锚定（Asset Anchoring）
**功能定位**：防空气化币

**机制说明**：
- AIXC 产生必须有真实车辆资产
- 收益来自长期真实使用

### 2.2 中层：行为贡献（Behavior Mining）
**功能定位**：防短期割韭菜

**机制说明**：
- 收益分期解锁需贡献一次性

### 2.3 上层：收益释放（Reward Distribution）
**功能定位**：防通盘炒作

**机制说明**：
- 通过奖励可调、节奏控制与规则约束，确保动态外在不间断于平的全车、稳定正可叮转移进行
- 不违背现实逻辑发逻辑，优先系统稳定与长期数据价值

### 2.4 周向：风险与治理（Risk & Governance）
**功能定位**：设计边界

### 2.5 斜面：设计边界（Design Boundary）
**功能定位**：不违背现实逻辑

## 3. 参与角色 & 行为路径（T3 Roles & Behavior Paths）

### 3.1 角色类型

#### 🔵 生态共建者（Ecosystem Builders）
- **需资质**
- **行为特征**：行为产生数据；按资按权；流动性入口
- **激励方式**：弹性激励

#### 🟢 行为贡献者（Behavior Contributors）
- **不需资质**
- **行为特征**：行为产生数据权；按资按权叠加分红
- **激励方式**：稳定释放

#### 🟡 退出/高车（Exit/High-value Vehicles）
- **行为特征**：一次性收益释放；流动性出口
- **激励方式**：可控调节

## 4. 激励释放逻辑链（T4 Incentive Release Logic）

### 4.1 核心机制流程

#### 📍 流程（Process Flow）
```
产生 → 分配 → 解锁 → 上限 → 异常收益
```

#### 📍 释放方式（Release Method）
- **不直接给用户**，先进入配额池
- **防过度发放**

#### 📍 分配方式（Allocation Method）
- **前行为需验证再分红**
- **受限真实使用**，冷启动+积累
- **越早越的收益更多**，后进入需支配要后排

#### 📍 时间机制（Time Mechanism）
- **时间锁定机制**
- **日上限**，每日奖励布需按比例接收
- **退出机制**

#### 📍 异常处理（Anomaly Detection）
- **异常行为检测系统**被共享监督行动可信与检坏
- 避免其费分高额分配，保障真实用户获得公平激励

#### 📍 参数可调（Parameter Adjustability）
- **核心公共**，可根据生态家庭条调
- 更长观照所叫规则提供

#### 📍 长期保护（Long-term Protection）
- 高资后可叮收据
- 真获收系

#### 📍 退出机制（Exit Mechanism）
- 退出行为不会釜主长载额测
- 避免短对出密密利

## 5. 核心激励机制详解

### 5.1 车辆上链（Vehicle Onboarding）

**启动期策略**：高激励 + 行为竞争

**用户路径**：
```
车 → 1. 卖车；2. 里程 
    ===买 AIXC (20,000)==> 
    1. 卖车+？；2. 里程+？；3. 年mint 质押收益
```

#### 5.1.1 Mint 公式

```
Mint_AIXC = min(
    max(Vehicle_USD_Value * Collateral_Rate / AIXC_Price, MinCap),
    MaxCap
)
```

**参数说明**：
- `Vehicle_USD_Value`：车辆美元估值
- `Collateral_Rate`：抵押率（项目组设定 = 0.3，即 30%）
- `AIXC_Price`：AIXC 当前价格
- `MinCap`：最小铸造量（项目组设定 = 1000 AIXC）
- `MaxCap`：最大铸造量（项目组设定 = 4000 AIXC）

**项目组参数设置**：
```
Collateral_Rate = 0.3
minCap/maxCap = 1000/4000
年释放 = 4/5 AIXC
总日释放 = 1,095,890.41 AIXC/天
phaseRatio = 0.8
```

**设计目的**：
- 控制车辆初始价值
- 车辆持有期间不可卖（锁定机制）

#### 5.1.2 案例计算

**用户示例**：

| 用户 | 车价（USD） | 年释放 AIXC | 车辆铸造时获得的 Mint_AIXC | 实际 Mint_AIXC | 备注 |
|------|-----------|------------|---------------------------|----------------|------|
| A | 10,000 | 2,000 | 30,000 | 30,000 | - |
| B | 12,000 | 10,000 | 36,000 | 36,000 | - |
| C | 20,000 | 0 | 60,000 | 40,000 | 触顶 |
| D | 8,000 | 0 | 24,000 | 24,000 | - |

### 5.2 车辆上链质押收益（20%）

#### 5.2.1 基本设定
- **占比**：占总激励的 20%
- **释放方式**：分三个月每天线性释放 AIXC 量
- **计算依据**：全网贡献度

#### 5.2.2 完整公式

```
Weight(i,day) = (0.8·Stake)·Mileage(i,30d)

Reward(i,day) = min(Poolday_user, pooldayConst) · Weight(i,day) / SUM(Weight(i,day))

phaseRatio(DayIndex) = {
    0.8,  1≤DayIndex≤90
    0.5,  91≤DayIndex≤180
    0.3,  DayIndex>181
}

Cap(i,day) = phaseRatio(DayIndex)·Stake / 90

Reward(i,day) = min(Reward(i,day), Cap(i,day))
```

**公式说明**：
- **权重计算**：质押权重（0.8 × Stake）× 最近30天里程
- **奖励池分配**：根据用户权重占比分配每日池子
- **分阶段衰减**：
  - 前 90 天：释放 80%
  - 91-180 天：释放 50%
  - 181 天后：释放 30%
- **每日上限**：防止单日集中释放

#### 5.2.3 数据提供者（不质押）

**说明**：下一个季度的 base 量是上一个季度质押用户平均量的 30%

**公式**：
```
baseStakingCount = avg(sum(usersStakingCount)) * 30%
```

## 8. SC（Supply Coefficient）多维调节系统

SC（Supply Coefficient，供给系数）是一个综合性调节参数，影响用户的 Mint 权限、行为挖矿奖励以及每日产币上限。

### 8.1 Mint 权限与额度由 SC 决定（SC_Multiplier）

| SC 区间 | SC_Multiplier | 说明 |
|---------|---------------|------|
| SC < 1.0 | 0.5 | 低信用，不让Mint，降低风险 |
| 1.0 ≤ SC < 3.0 | 1 | 可mint一辆车 |
| 3.0 ≤ SC < 6.0 | 1.3 | 可mint2辆车 |
| SC ≥ 6.0 | 1.6 | 可 mint 多车 & 更高 Collateral Rate |

**设计目的**：
- 根据用户信用/贡献度，动态调整 Mint 权限
- 防止低信用用户滥用系统

### 8.2 SC 对行为挖矿（产币）的影响（SC_Boost）

| SC 区间 | SC_Boost | 说明 |
|---------|----------|------|
| SC < 1.0 | 0.8 | 低信用 → 降产出 |
| 1.0 ≤ SC < 2.5 | 1 | 标准产出 |
| 2.5 ≤ SC < 4.0 | 1.1 | 中级信用提升 |
| 4.0 ≤ SC < 6.0 | 1.25 | 高信用加成 |
| SC ≥ 6.0 | 1.5 | 超高信用 → 最高产能 |

**设计目的**：
- 奖励高贡献度用户
- 抑制低信用用户的收益

### 8.3 SC 对每日产币上限（Cap）影响（SC_Cap）

| SC 区间 | SC_Cap | 说明 |
|---------|--------|------|
| SC < 1 | 0.5 | 降50% |
| 1 ≤ SC < 3 | 1 | 标准产出 |
| 3 ≤ SC < 6 | 1.3 | 提升 |
| SC ≥ 6 | 1.8 | 高信用加成 |
| >6.0 | 1.5 | 超高信用 → 最高产能 |

**设计目的**：
- 通过调节每日上限，平滑奖励释放
- 防止单日集中抛压

## 11. 完整用户案例：A用户收益模型

**用户信息**：
- 车价（USD）：20,000
- AIXC当前价格：0.1
- Collateral_Rate：0.3
- 计算 Mint 量：60,000 AIXC
- 实际 Mint（受 MaxCap 限制）：40,000 AIXC

**A用户 Mint 计算**：
```
Mint_AIXC = min(
    max(20000 * 0.3 / 0.1, 1000),
    4000
) = min(60000, 4000) = 40,000 AIXC（触顶限制）
```

**收益构成**：
1. **质押收益（20%）**：根据质押时长和里程动态释放
2. **里程挖矿（80%）**：根据实际行驶里程获得奖励
3. **卖车收益**：根据持有时长和贡献度获得额外奖励

## 12. 关键设计亮点

### 12.1 多层防护机制
1. **资产锚定**：车辆价值作为底层支撑（100亿总量）
2. **行为验证**：收益需通过真实行为获得
3. **分期释放**：防止集中抛压（phaseRatio 衰减）
4. **动态调节**：SC 系数多维度调节
5. **异常检测**：AI 识别不合理行为

### 12.2 激励平衡
- **早期参与者**：高额补贴，快速冷启动（前5年每年4亿）
- **长期贡献者**：持续奖励，Account Factor 加成
- **高价值资产**：Vehicle Value 系数加成
- **活跃用户**：SC_Boost 提升产出

### 12.3 可持续性
- **前期**：代币激励为主，吸引用户
- **中期**：代币激励 + 真实收益并重
- **后期**：真实收益为主，代币激励为辅
- **20年后**：完全依靠真实服务收入
